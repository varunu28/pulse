# Design notes

Long-form architectural rationale for Pulse subsystems where the choice
wasn't obvious. These are the documents you read once before you make a
PR that changes the subsystem.

| Note | Subsystem |
|---|---|
| [Multi-tenant context](multi-tenant-context.md) | [features/multi-tenant.md](../features/multi-tenant.md) |

More design notes will be added as subsystems evolve. If you're proposing a
non-trivial change to an existing subsystem, the convention is "post the
design note as the first PR, then the implementation."
